The problem

Sometimes you just want to read a markdown file. Not edit it, not file it in a vault, not preview it in the corner of an IDE. Read it. There is no good place to do that. Obsidian wants the file in your graph. VSCode's preview is fine, but the rest of VSCode is right there pulling focus. Browsers do not render markdown. Quick Look mangles code blocks.

And the files themselves are not pure markdown anymore. LLM outputs mix prose with embedded HTML, XML, JSON, and code: system and user tags in a prompt library, detail blocks in a spec, raw SVG in a diagram. A reader that strips or fumbles those is not reading the document. It is reading a sanitized version of it.

What I built

A small native macOS app that does one thing well: open the file, read the file, close the window.

  • Honest rendering. Full prose layout and syntax-highlighted code, with the embedded markup kept visible and legible in place instead of stripped out.
  • Reading and source views. One shortcut toggles between the rendered output and the raw text, for any text file.
  • Visual care in the reader. The rendered view is styled by CSS templates you can swap, write, and share. Describe the look you want to a model, paste the CSS, and read every document in your own look rather than a fixed theme.
  • Optional model assist. Connect an Anthropic or OpenAI-compatible key (stored in the macOS Keychain) and iterate on templates by describing changes in plain English. Entirely optional. The app works fully without it.
  • Real and shippable. Tauri 2 on the system webview, so the footprint is tiny, with a native menu bar, keyboard shortcuts, auto-update, signed and notarized so Gatekeeper lets it run cleanly.

What it made possible

A focused place to actually read what agents write, the way it was written, in a visual treatment you control. The next step is sandboxed rendering of the HTML artifacts agents are starting to produce instead of markdown, specs and explainers and throwaway editors, which deserve a focused reader too, not a browser tab buried among twenty others.

Why it matters

As more of our reading becomes machine-generated, where and how we read it stops being an afterthought. View Anew is a bet that the care people put into a terminal prompt or an editor theme belongs in the reader as well, and it is proof that I will ship a finished, notarized product to make that point rather than just describe it.
